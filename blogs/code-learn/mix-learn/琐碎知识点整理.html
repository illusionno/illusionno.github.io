    # 1. call, apply,bind

    call第二个参数是参数列表，apply是参数数组 🔔区别：call, apply在调用函数后会立即执行，bind调用函数后会返回一个新的函数。

    # 2. for ...in / of

    for..in数组得到索引，for..of数组得到值

    # 3. 数组. indexOf(元素)

    找到数组中第一个符合元素的下标，找不到为-1

    # 4. 防抖、节流

    防抖：在事件被触发的n秒后再执行回调，如果在这n秒中又被触发，则重新计时。 应用场合：搜索框的联想功能，验证码，表单提交，resize事件等 🔔：规定在一个单位时间内，只触发一次函数。如果触发多次，则无效 应用场合：scroll事件，input事件，播放事件

    # 5. 行内元素margin只能设置左右，padding上下左右都有效

    # 6. 数组的splice

    数组的splice 会改变原数组

     var arr = [2,4,6,7,8,9]; 
 var n = arr.splice(2);
 console.log(n);//[6,7,8,9] 从该字符截取到最后
 console.log(arr);//[2,4] 剩下的
-----------------------------
var n = arr.splice(2,3);
console.log(n);//[6, 7, 8] 从下标位置为2开始截取3个数
console.log(arr);//[2,4,9] 从下标2开始删除3个
-----------------------------
var n = arr.splice(2,0,5);
console.log(arr);//[2, 4, 5, 6, 7, 8, 9]

    # 19. 轮询

    • 短轮询：浏览器每隔一段时间向服务器发送http请求，服务器端在收到请求后，不论是否有数据更新，都直接进行响应（基于http）

    • 长轮询：当服务器收到客户端发来的请求后,服务器端不会直接进行响应，而是先将这个请求挂起，然后判断服务器端数据是否有更新。如果有更新，则进行响应，如果一直没有数据，则到达一定的时间限制(服务器端设置)才返回（基于http）

    • 长连接（SSE）：不需要客户端发送请求，可以实现只要服务器端数据有更新，就可以马上发送到客户端（基于http）

    • WebSocket ：首先需要在客户端和服务器端建立起一个连接，这部分需要http。连接一旦建立，客户端和服务器端就处于平等的地位，可以相互发送数据，不存在请求和响应的区别。

      实现服务端自动推送: SSE，WebSocket **补充：**基于http协议的连接需要事先建立连接

    # 20. overflow: auto

    当元素超出设定的高度时，会出现滚动条. 父元素设置flex， 左边设置width，右边flex: 1, 左边超出部分使用overflow: hidden, 右边超出部分使用overflow: auto
