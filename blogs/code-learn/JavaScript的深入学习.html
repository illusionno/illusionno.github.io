    JavaScript的深入学习

    zhou 2022/12/16 学习笔记

    JavaScript 是编译语言。

    # 1. 编译的三个步骤

    • 分词/词法分析（Tokenizing/Lexing） ： 将字符串分解成（对编程语言来说）有意义的代码块 ， var a = 2；会被分解成 词法单元：var、a、=、2 、;
    • 解析/语法分析（Parsing） ：转为抽象语法树 （AST）
    • 代码生成： 将 AST 转换为可执行代码的过程

    # 2. 理解作用域

    引擎：从头到尾负责整个 JavaScript 程序的编译及执行过程。
    编译器: 负责语法分析及代码生成等。 作用域：负责收集并维护由所有声明的标识符（变量）， 确定当前执行的代码对这些标识符的访问权限 。 编译器的术语 ： 当变量出现在赋值操作的左侧时进行LHS 查询，出现在右侧查找变量时进行 RHS 查询

    console.log(a); //RHS 引用
a = 2; //LHS 引用

    # 4. 闭包

    当函数可以记住并访问所在的词法作用域，即使函数是在当前词法作用域之外执行，这时 就产生了闭包。
    setTimeout 中传入回调函数就是闭包

    for (var i = 1; i <= 5; i++) {
  setTimeout(function timer() {
    console.log(i);
  }, i * 1000); //5个6

    模块的特征：提供暴露的接口，让外部可以调用。

      1. 必须有外部的封闭函数，该函数必须至少被调用一次
      1. 封闭函数必须返回至少一个内部函数
    function CoolModule() {
  var something = "cool";
  var another = [1, 2, 3];

  function doSomething() {
    console.log(something);
  }
  function doAnother() {
    console.log(another.join(" ! "));
  }
  return {
    doSomething: doSomething,
    doAnother: doAnother,
  };
}
var foo = CoolModule();
foo.doSomething(); // cool
foo.doAnother(); // 1 ! 2 ! 3

    # 5. this

    this 提供了一种更优雅的方式来隐式“传递”一个对象引用 。 this 的绑定和函数声明的位置没有任何关系，只取决于函数的调用方式。
    当一个函数被调用时，会创建一个执行上下文，包含函数在哪里被调用（调用栈）、函数的调用方法、传入的参数等信息。

    四种绑定方法：

    • 默认绑定： 在非 strict mode 下时，默认绑定 this 指向全局变量
    function foo() {
  console.log(this.a); //2
}
var a = 2;
