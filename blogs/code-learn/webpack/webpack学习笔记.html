    # webpack是什么？有什么作用？

    Webpack是一个开源的前端打包工具。 将浏览器不能识别的代码编译成浏览器可以识别的代码，比如在浏览器中不能识别ES6的模块化语法（babel转成es5）。除此之外，打包工具还能压缩代码、做兼容性处理、提升代码性能。

    # webpack配置文件的作用？

    配置入口、出口、插件等。 webpack.config.js 只能放在项目根目录下。运行指令：npx webpack

    # 💜5大核心概念

    1. entry（入口）

    指示 Webpack 从哪个文件开始打包

    1. output（输出）

    指示 Webpack 打包完的文件输出到哪里去，如何命名等

    1. loader（加载器）

    webpack 本身只能处理 js、json 等资源，不能识别的文件要借助 loader， 多个loader从右向左执行。 use选项可以是多个loader，loader选项只能一个。（代码）

    1. plugins（插件）

    扩展 Webpack 的功能

    1. mode（模式）：development（开发模式） / production（生产模式） 在开发模式下的主要做些什么:
    2. 编译代码，使浏览器能识别运行 开发时我们有样式资源、字体图标、图片资源、html 资源等，webpack 默认都不能处理这些资源，所以我们要加载配置来编译这些资源
    3. 检查代码，规范编码风格

    --mode 指定模式（环境） --config 指定配置文件

    Webpack的整体布局:

    module.exports = {
  // 入口
  entry: "",
  // 输出
  output: {},
  // 加载器
  module: {
    rules: [],
  },
  // 插件
  plugins: [],
  // 模式
  mode: "",
};

    # 图片处理?

    Webpack4：使用到file-loader和url-loader, url-loader可以将小图片转换成base64字符串，从而减少发送请求数（体积会变大） Webpack5：将两个 Loader 功能内置到 Webpack 里了，我们只需要简单配置即可处理图片资源

    # 提升开发体验-SourceMap

    SourceMap是一个源文件和编译后文件的映射，当代码报错后会准确定位到源文件里的位置，否则会定位到编译后文件里。 使用：直接配置devtool属性

    • 开发模式：cheap-module-source-map
      • 优点：打包编译速度快，只包含行映射
      • 缺点：没有列映射
    module.exports = {
  // 其他省略
  mode: "development",
  devtool: "cheap-module-source-map",
};
